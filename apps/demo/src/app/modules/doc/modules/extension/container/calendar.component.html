Styles are not required, as the dbx-web library already builds custom/modified styling for use.

All style colors are used as css variables, allowing for easy overwriting of values.

Timezone Tester

The dateScheduleRangeField() uses the timezone to change the start and end dates of the output. It treats all input as if it is already in that timezone. Timezone values returned will be from the start of that day's timezone.

Date schedules with a filter do not apply the timezone offset , as the filter's start date already describes which timezone they are in per the DateSchedule's Timing behavior. The timezone string is passed as a visual output however.

Customizing Dialog Footer

The footer can be customized globally using the DEFAULT_DBX_SCHEDULE_SELECTION_CALENDAR_DATE_POPUP_CLOSE_CONFIG_TOKEN to provide a custom close config.
